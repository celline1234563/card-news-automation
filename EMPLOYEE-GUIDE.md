# 직원용 카드뉴스 설정 가이드

> 이 문서는 Claude Code가 직원과 대화할 때 참고하는 가이드입니다.
> 직원이 Claude Code를 열면 이 파일을 기반으로 안내합니다.

---

## Claude Code 사용법 (직원용)

### 시작하기

1. 터미널에서 이 프로젝트 폴더를 열어주세요
2. `claude` 를 입력하면 Claude Code가 시작됩니다
3. **한국어로 편하게 말하면 됩니다**

### 할 수 있는 작업

직원은 아래 작업을 Claude Code에 자연어로 요청할 수 있습니다:

| 하고 싶은 것 | 이렇게 말하세요 |
|-------------|---------------|
| 헤드라인 말투 변경 | "진학학원 헤드라인을 좀 더 부드럽게 바꿔줘" |
| 금지 표현 추가 | "올인원에서 '폭발적' 이란 표현 금지시켜줘" |
| 색상 변경 | "톡톡 학원 메인 색상을 좀 더 진한 주황으로 바꿔줘" |
| 글씨 크기 조절 | "진학학원 헤드라인 글씨를 좀 줄여줘" |
| 감정 곡선 조절 | "1번 카드가 너무 자극적인데 좀 부드럽게" |
| 타겟 학교 추가 | "올인원에 불곡중학교 추가해줘" |
| 페르소나 수정 | "진학 타겟 학부모 연령을 45세로 올려줘" |
| 배경 이미지 스타일 변경 | "hook 카드 배경을 매거진풍으로 바꿔줘" |
| 이미지 무드 변경 | "배경 이미지가 너무 어두운데 밝게 해줘" |
| 미리보기 실행 | "진학학원으로 미리보기 한번 돌려봐" |
| 변경 반영 | "지금까지 바꾼 거 저장하고 배포해줘" |

### 작업 흐름

```
1. Claude Code에 수정 요청 (한국어)
2. Claude가 파일 수정
3. "미리보기 돌려봐" → 결과 확인
4. 만족하면 "저장하고 배포해줘" → git push → 자동 배포
5. Google Chat에 변경 전/후 미리보기 도착
```

## 파일 구조 (직원이 알아야 할 것만)

```
config/
├── brand/
│   ├── {학원}-strategy.md    ← 브랜드 전략 (타겟, 페르소나, 콘텐츠 방향)
│   ├── {학원}-voice.md       ← 말투 가이드 (헤드라인 공식, DO/DON'T)
│   └── emotion-curve.json    ← 감정 곡선 (공용)
prompts/
│   └── imager-system.txt     ← 배경 이미지 스타일 (무드, 스타일 키워드)
├── tokens/
│   └── {학원}-tokens.json    ← 디자인 토큰 (색상, 글씨, 그림자)
└── academies.json            ← 학원 기본 설정
```

**학원 키**: ollinone(올인원), jinhak(진학), toktok(톡톡)

## Claude Code 응대 지침

### 대화 시작 시

직원이 처음 대화를 시작하면:
1. 어떤 학원 담당인지 물어보세요 (올인원 / 진학 / 톡톡)
2. 무엇을 수정하고 싶은지 물어보세요
3. 해당 파일의 현재 내용을 보여주고 어떻게 바꿀지 확인하세요

### 수정 작업 시

1. **반드시 수정 전 내용을 보여주고** 확인 받은 후 수정하세요
2. 수정 후에는 바뀐 부분을 요약해서 알려주세요
3. "미리보기 돌려볼까요?" 라고 물어보세요

### 미리보기 실행

```bash
node scripts/preview-bot.js --academy {학원키} --after-only
```

### 저장 & 배포

```bash
git add config/ prompts/
git commit -m "{학원명} 설정 업데이트: {변경 내용}"
git push
```

### 주의사항

- `tokens.json`의 메인 컬러를 바꿀 때는 `academies.json`의 theme도 같이 바꿔야 합니다
- `emotion-curve.json`은 모든 학원 공용입니다
- 코드 파일(`.js`)은 절대 수정하지 마세요 — 설정 파일만 수정합니다
- 프롬프트 파일(`prompts/*.txt`)도 수정 가능하지만, 신중하게 해주세요

## 회의록 등록 워크플로우 (직원이 가장 자주 쓰는 작업)

**상황**: 사장님이 원장님 회의록을 보내주시고, "노션에 등록해줘" 라고 함.

**직원이 입력하는 형식** (이 메시지를 그대로 Claude Code에 붙여넣으면 됨):

```
회의록 등록해줘.
학원: 진학        ← 진학 / 올인원 / 톡톡 셋 중 하나

[회의록 본문 — 길이 제한 없음, 그냥 받은 그대로]
```

### Claude Code가 해야 할 동작 (이 가이드를 읽는 너에게)

직원이 위 형식으로 메시지를 보내면, 아래 순서로 처리해.

1. **학원 키 매핑**: `진학 → jinhak`, `올인원 → ollinone`, `톡톡 → toktok`. 다른 단어가 오면 다시 물어봐.

2. **주제 한 줄 뽑기**: 회의록 본문에서 핵심 주제를 한국어 15자 이내로 한 줄 뽑아내. 예: "중3에 시작해야 하는 이유", "여름방학 특강 모집". 그리고 **직원에게 "주제를 '___'로 잡으면 될까요?" 라고 한 번 확인**받아.

3. **노션에 등록**: 직원이 OK 하면 아래 한 줄짜리 인라인 스크립트로 등록해. 회의록 본문은 stdin으로 흘려넣어 (특수문자 escape 문제 회피).

   ```bash
   cat <<'INTAKE_EOF' | node --input-type=module -e "
   import { createIntake } from './agents/notion-connector.js';
   const content = await new Promise(r => { let d=''; process.stdin.on('data', c => d+=c); process.stdin.on('end', () => r(d)); });
   const result = await createIntake({ academyKey: 'jinhak', topic: '중3에 시작해야 하는 이유', content });
   console.log(JSON.stringify(result));
   "
   [여기에 회의록 본문 그대로]
   INTAKE_EOF
   ```

   `academyKey`와 `topic`은 위 1, 2 단계에서 나온 값으로 치환해.

4. **응답**: 노션 URL과 제목을 직원에게 알려주고, "30초 안에 작은기획이 자동 생성됩니다. 10분쯤 후 노션 카드에 PNG가 첨부됩니다." 라고 안내해.

### 주의

- **createIntake가 자동으로 채우는 것**: 제목 `[학원 월-번호]`, 상태 `기획 착수`, 제작 포맷 `카드뉴스`. 번호는 같은 학원/월 카드 개수를 세서 +1 자동 채번.
- **회의록 본문은 절대 다듬지 마**: 받은 그대로 노션에 들어가야 researcher가 정확한 맥락으로 작은기획을 짤 수 있음.
- **여러 학원이 한 회의록에 섞여있으면**: 학원별로 나눠서 `createIntake`를 학원 수만큼 호출. 직원에게 "회의록을 학원별로 나눠 등록할게요" 라고 한 번 알려주고 진행.
- **에러가 나면**: 에러 메시지 그대로 직원에게 보여주고 멈춰. `.env`의 `NOTION_API_KEY` / `NOTION_DATASOURCE_ID` 누락이 가장 흔한 원인.
