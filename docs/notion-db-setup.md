# 노션 DB 속성 설정 가이드

## 1. 데이터베이스 속성

| 속성명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| 제목 | 제목 | O | `[학원명] 콘텐츠 제목` 형식 |
| 상태 | multi_select | O | 파이프라인 상태 관리 |
| 메인 키워드 | rich_text | - | SEO 키워드 |
| 제작 포맷 | multi_select | - | "카드뉴스" 필터용 |
| 학원정보 | multi_select | - | 현재 제목 prefix로 판별 |
| 비고 | rich_text | - | 메모용 |

## 2. 상태 값 (multi_select 옵션)

| 상태 | 트리거 | 자동 동작 |
|------|--------|----------|
| 기획 착수 | 사용자가 설정 | → Claude 리서치 + 카드 10장 기획 → 기획 컨펌 |
| 기획 컨펌 | 자동 설정 | 대기 (사용자가 검토) |
| 제작 요청 | 사용자가 설정 | → 디자인 파이프라인 실행 → 디자인 1차 |
| 디자인 1차 | 자동 설정 | PNG 첨부 완료 |
| 디자인 수정 | 사용자가 설정 | @수정 댓글 감지 → 재기획 → 디자인 수정 완료 |
| 디자인 수정 완료 | 자동 설정 | 새 PNG 첨부 완료 |
| 게시완료 | 사용자가 설정 | 아카이브 |

## 3. 제목 형식

```
[올인원] 중3에 시작해야 하는 이유
[진학] 초등 영어 학원 선택 기준
[톡톡] 국제학교 준비 가이드
```

- `[학원명]` 접두어로 학원 자동 판별
- 올인원 → ollinone, 진학 → jinhak, 톡톡 → toktok

## 4. 댓글 활용

### 재료 전달
페이지 댓글에 참고자료, 통계, 원장님 메모 등을 작성하면 기획 시 자동으로 참조됩니다.

### 수정 요청
```
@수정 1번 카드 헤드라인을 더 자극적으로
@수정 전체적으로 톤을 부드럽게
```

- `@수정`으로 시작하는 댓글만 수정 지시로 인식
- 상태를 "디자인 수정"으로 변경하면 자동 처리

## 5. 초기 설정

1. 노션 인테그레이션 생성 (https://www.notion.so/my-integrations)
2. 데이터베이스에 인테그레이션 연결
3. `.env`에 설정:
   ```
   NOTION_API_KEY=ntn_xxxxx
   NOTION_DATABASE_ID=데이터베이스_UUID
   NOTION_DATASOURCE_ID=데이터소스_UUID
   ```
4. `node poller.js`로 폴링 시작
