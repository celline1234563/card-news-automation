【절대 규칙 1: 텍스트 번역 금지】
- 입력된 한국어 텍스트를 절대 영어로 번역하지 마세요
- headline, subtext, stat_label 등 모든 텍스트를 입력 그대로 HTML에 출력하세요
- "선택 기준" → "Selection Criteria" 이런 변환 절대 금지
- 학원명도 한국어 그대로 출력: "우리수학학원" → "우리수학학원"

【절대 규칙 2: 한글 폰트 필수 로드】
- <head>에 반드시 아래 태그를 포함하세요:
  <link href="https://fonts.googleapis.com/css2?family=Noto+Sans+KR:wght@400;700;900&display=swap" rel="stylesheet">
- body, 모든 요소에 font-family: 'Noto Sans KR', sans-serif; 적용
- 다른 폰트를 사용하면 한글이 깨집니다. Noto Sans KR만 사용하세요

【절대 규칙 3: CSS 변수만 사용, hex값 직접 사용 금지】
- 색상은 반드시 var(--color-primary) 형태로만 사용하세요
- #2563EB, #EF4444 같은 hex값을 CSS에 직접 쓰지 마세요
- HTML 텍스트 콘텐츠에 hex값이나 CSS 변수명이 보이면 안 됩니다
- 제공된 :root 변수 블록을 <style> 최상단에 그대로 복사하세요

【절대 규칙 4: 캔버스 고정 + 콘텐츠 보호】
- body { width: 1080px; height: 1350px; overflow: hidden; margin: 0; }
- 모든 콘텐츠는 반드시 1080x1350px 영역 안에 배치하세요
- 이미지가 텍스트를 가리면 안 됩니다
- 배경 이미지는 z-index 가장 뒤, 텍스트는 가장 앞
- position: absolute 사용 시 텍스트가 영역 밖으로 나가지 않도록 주의

【절대 규칙 5: 배경 이미지 위 텍스트 가독성】
- 배경 이미지 위에 텍스트를 올릴 때 반드시 반투명 오버레이를 추가하세요
- 예시: background: rgba(0, 0, 0, 0.5) 또는 background: rgba(255, 255, 255, 0.85)
- 오버레이 없이 이미지 위에 직접 텍스트를 올리면 가독성 실패입니다

## HTML 구조 (이 순서대로 작성)
```
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <link href="https://fonts.googleapis.com/css2?family=Noto+Sans+KR:wght@400;700;900&display=swap" rel="stylesheet">
  <style>
    /* 1. 제공된 :root CSS 변수 블록 그대로 복사 */
    /* 2. body { width:1080px; height:1350px; overflow:hidden; font-family:'Noto Sans KR',sans-serif; } */
    /* 3. 나머지 스타일 */
  </style>
</head>
<body>...</body>
</html>
```

## <em> 태그 강조 스타일
- emphasis_style이 highlight → em { background: var(--color-highlight); font-style: normal; padding: 2px 8px; }
- emphasis_style이 color → em { color: var(--color-primary); font-weight: 900; font-style: normal; }
- emphasis_style이 underline → em { border-bottom: 6px solid var(--color-accent); font-style: normal; padding-bottom: 2px; }

## 텍스트 보호 + 줄바꿈 규칙
- 모든 텍스트 컨테이너에 충분한 padding (최소 60px)
- headline font-size: 최대 48px (텍스트가 길면 36px까지 축소)
- subtext font-size: 24~28px
- 헤드라인 한 줄 최대 16자. 초과 시 자동 줄바꿈 되도록 max-width 설정
- word-break: keep-all 반드시 적용 (한국어 단어 중간에서 잘리면 안 됨)
- line-height: 1.4 이상 유지 (줄 간격이 좁으면 가독성 저하)
- 텍스트가 잘리거나 카드 밖으로 나가면 실패

## 10장 전체 컬러 일관성 규칙 (매우 중요)
- 포인트 컬러: var(--color-primary) 하나만 사용
- 배경색: var(--color-background) 또는 #ffffff(흰색)만 사용
- 강조 텍스트(em 태그, stat 숫자)만 var(--color-accent) 허용
- 카드마다 컬러 테마가 달라지면 안 됨 — 10장이 하나의 시리즈로 보여야 함
- 오버레이는 반드시 var(--color-primary) 기반 rgba로 통일
  예: rgba(37, 99, 235, 0.85) 대신 → 어두운 오버레이가 필요하면 rgba(26, 26, 46, 0.7) 사용
- var(--color-secondary)는 아이콘 배경, 작은 배지에만 제한적으로 사용

## 카드 타입별 레이아웃 (강제 지정)
카드의 type 값에 따라 아래 레이아웃 구조를 반드시 따르세요:
- hook → 상단 2/3 이미지 또는 그라디언트 풀블리드 + 하단 1/3에 텍스트 배치. 임팩트 있는 큰 헤드라인
- problem/empathy → 좌측에 var(--color-primary) 세로 사이드바(8px) + 우측에 텍스트 본문
- data → 중앙에 stat 숫자 초대형(100~120px, var(--color-accent)) + 하단에 stat_label과 headline
- insight → 상단 아이콘 + 중앙 텍스트, 깨끗한 카드형 레이아웃
- solution → 체크리스트 스타일 또는 번호 매긴 스텝형 레이아웃
- example → 비교 카드(before/after) 또는 인용문 스타일
- summary → 핵심 포인트 3~4개를 불릿 리스트로 정리
- cta → var(--color-primary) 풀배경 + 흰색 텍스트 + 둥근 CTA 버튼(흰 배경, primary 텍스트)
- 위 구조를 기본으로 하되, usedLayouts에 같은 구조가 있으면 변형해서 사용
