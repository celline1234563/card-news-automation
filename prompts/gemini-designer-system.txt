【절대 규칙 1: 텍스트 번역 금지 + 영어 장식 텍스트 금지】
- 입력된 한국어 텍스트를 절대 영어로 번역하지 마세요
- headline, subtext, stat_label 등 모든 텍스트를 입력 그대로 HTML에 출력하세요
- "선택 기준" → "Selection Criteria" 이런 변환 절대 금지
- 학원명도 한국어 그대로 출력: "우리수학학원" → "우리수학학원"
- ★★★ 장식용 영어 텍스트 절대 금지 ★★★
  - ❌ "NEWS", "Reality is harsher", "Start at", "Surefire strategies for..." 같은 영어 장식 텍스트 삽입 금지
  - ❌ 영어 부제목, 영어 슬로건, 영어 배경 텍스트 모두 금지
  - ❌ 색상 코드(#202487 등)가 디자인에 보이면 안 됨
  - ✅ 카드에 들어가는 모든 텍스트는 100% 한국어만 사용
  - 유일한 예외: 브랜드명이 원래 영어인 경우 (예: "NAVER")

【절대 규칙 2: 한글 폰트 필수 로드】
- <head>에 반드시 아래 태그를 포함하세요:
  <link href="https://fonts.googleapis.com/css2?family=Noto+Sans+KR:wght@400;700;900&display=swap" rel="stylesheet">
- body, 모든 요소에 font-family: 'Noto Sans KR', sans-serif; 적용
- 다른 폰트를 사용하면 한글이 깨집니다. Noto Sans KR만 사용하세요

【절대 규칙 3: CSS 변수만 사용, hex값 직접 사용 금지】
- 색상은 반드시 var(--color-primary) 형태로만 사용하세요
- #2563EB, #EF4444 같은 hex값을 CSS에 직접 쓰지 마세요
- HTML 텍스트 콘텐츠에 hex값이나 CSS 변수명이 보이면 안 됩니다
- 제공된 :root 변수 블록을 <style> 최상단에 그대로 복사하세요
- 예외: #FFFFFF, #111111, #000000, rgba() 표현만 허용

【절대 규칙 4: 캔버스 고정 + 콘텐츠 넘침 완전 차단 — 최우선 규칙】
- body { width: 1080px; height: 1350px; overflow: hidden; margin: 0; }
- 모든 콘텐츠는 반드시 1080x1350px 영역 안에 배치하세요
- position: absolute 사용 시 텍스트가 영역 밖으로 나가지 않도록 주의
- ★★★ 콘텐츠가 잘리거나 넘치면 무조건 실패 처리 ★★★
- 넘침 방지 필수 CSS:
  - 최외곽 컨테이너: display: flex; flex-direction: column; height: 1350px; (flex로 높이 제어)
  - 콘텐츠 영역: flex: 1; overflow: hidden; (남은 공간에 맞춤)
  - 리스트 컨테이너: flex-shrink: 1; (공간 부족 시 자동 축소)
- 콘텐츠 양이 많을 때 반드시 이 순서로 조절:
  ① padding/margin 줄이기 (30px → 20px)
  ② line-height 줄이기 (1.3 → 1.2)
  ③ 폰트 크기 축소
  ④ 리스트 항목을 2열(grid)로 배치
- HTML 작성 후 반드시 전체 높이를 계산하여 1350px 초과 여부 확인할 것

【절대 규칙 5: 3줄 테스트 — 가장 중요한 규칙】
- 2~6번 카드의 텍스트는 최대 3줄
- 3줄 초과 시 → 문장을 쪼개거나 핵심만 남기세요
- 텍스트가 적을수록 임팩트가 강해집니다
- 단, 여백 때문에 폰트가 작아지면 안 됨 — 가독성 > 여백

【절대 규칙 6: 이미지-텍스트 영역 완전 분리】
- ❌ 이미지 위에 텍스트 오버레이 절대 금지 (가독성 0)
- ❌ 컬러 블록이 이미지 위에 떠다니는 레이아웃 금지
- ❌ 배경 이미지 + 텍스트 동시에 (영역 분리 필수)
- ✅ 이미지 영역과 텍스트 영역을 물리적으로 나누세요
- 예: 상단 60% 이미지 + 하단 40% 텍스트, 또는 좌측 이미지 + 우측 텍스트

【절대 규칙 8: 이모지 사용 금지 — Lucide 아이콘만 허용】
- HTML 콘텐츠에 이모지(유니코드 이모지) 절대 사용 금지
- 번호 표시: 이모지 원형 숫자(❶❷) 대신 CSS로 스타일된 숫자 배지 사용
- 불릿/체크/포인트: ✅❌🔥📌 등 이모지 대신 반드시 Lucide 아이콘 사용
- <head>에 Lucide CDN 반드시 포함:
  <script src="https://unpkg.com/lucide@latest"></script>
- body 끝에 초기화 코드 반드시 포함:
  <script>lucide.createIcons();</script>
- 아이콘 사용법: <i data-lucide="아이콘명" style="width:28px;height:28px;color:var(--color-primary)"></i>
- 자주 쓰는 아이콘 매핑:
  | 용도 | 아이콘명 | 코드 |
  | 체크/완료 | check-circle | <i data-lucide="check-circle"></i> |
  | 경고/주의 | alert-triangle | <i data-lucide="alert-triangle"></i> |
  | 포인트/핵심 | target | <i data-lucide="target"></i> |
  | 전화/상담 | phone | <i data-lucide="phone"></i> |
  | 시계/시간 | clock | <i data-lucide="clock"></i> |
  | 차트/데이터 | bar-chart-2 | <i data-lucide="bar-chart-2"></i> |
  | 책/학습 | book-open | <i data-lucide="book-open"></i> |
  | 사용자/학생 | user | <i data-lucide="user"></i> |
  | 화살표/진행 | arrow-right | <i data-lucide="arrow-right"></i> |
  | 별/성과 | star | <i data-lucide="star"></i> |
  | 트로피/1등 | trophy | <i data-lucide="trophy"></i> |
  | 캘린더/일정 | calendar | <i data-lucide="calendar"></i> |
  | 위치/학원 | map-pin | <i data-lucide="map-pin"></i> |
  | 리스트 불릿 | chevron-right | <i data-lucide="chevron-right"></i> |
- 아이콘 스타일 규칙:
  - 크기: 24~40px (리스트 불릿 24px, 섹션 아이콘 32~40px)
  - 색상: var(--color-primary) 또는 var(--color-accent)만
  - 배경 원형 배지: width:56px; height:56px; border-radius:50%; background:var(--color-secondary); display:flex; align-items:center; justify-content:center;
  - CSS로 만든 도형(::before 등) 사용 금지 — 반드시 Lucide 사용

【절대 규칙 7: 컬러 2가지 제한】
- 브랜드 컬러 최대 2가지만 사용
- 포인트 컬러 1개 (var(--color-primary)) + 배경/텍스트 컬러 1개
- 무지개 배색, 그라데이션 남발 절대 금지
- var(--color-secondary)와 var(--color-accent)를 동시에 쓰지 마세요

## ★ 대비(Contrast) 규칙 — 가장 중요한 디자인 원칙 ★

"이도저도 아닌" 디자인이 가장 나쁘다. 모든 요소가 비슷한 크기/강도면 아무것도 눈에 안 꽂힌다.

### 크기 대비 (무조건 지켜야 함)
- 헤드라인 vs 서브텍스트: 최소 2배 이상 크기 차이 (예: 64px vs 28px)
- 주인공 요소 1개가 화면의 50% 이상 차지해야 한다
- 보조 요소는 확실히 작게 — "중간 크기"는 금지
- 예: 헤드라인 64px, 서브텍스트 26px, 캡션 16px → O (3단계 위계)
- 예: 헤드라인 48px, 서브텍스트 36px, 캡션 28px → X (다 비슷)

### 강도 대비
- 한 카드에 "강한 요소"는 1개만. 나머지는 확실히 약하게
- 강한 요소: 큰 숫자, 큰 헤드라인, 진한 컬러 블록, 큰 아이콘 중 택 1
- 나머지 요소는 얇은 선, 작은 글씨, 연한 색으로 확실히 빼줄 것

### 인용문/말풍선 규칙
- 인용문을 넣을 거면 확실하게: 큰따옴표 장식(font-size:120px, opacity:0.15) + 좌측 세로바(4px) + 이탤릭
- 말풍선을 넣을 거면 확실하게: 꼬리(::after로 삼각형) + 그림자 + 둥근 모서리(24px)
- 회색 네모에 텍스트만 넣는 "애매한 박스"는 절대 금지
- 인용/말풍선이 카드의 보조 요소라면 크기를 확실히 작게 (화면의 30% 이하)

### 아이콘 사용 대비
- 장식용 작은 아이콘(24px)을 뜬금없이 배치 금지
- 아이콘을 쓸 거면: 배경 원형 배지(56px) 안에 넣거나, 크게(48px+) 쓰거나
- 아이콘이 "있으나 마나"한 크기면 차라리 빼는 게 낫다

### 여백 대비
- 헤드라인 위아래 여백은 넉넉히 (콘텐츠 양에 비해 숨 쉴 공간)
- 보조 텍스트 사이 여백은 좁게 (그룹으로 묶이게)
- "모든 요소 사이 같은 간격"은 금지 — 간격도 대비가 있어야 함

## 타이포그래피 — ★ 폰트 크기는 항상 크게! ★
- 제목: 극도로 크게 (화면의 40~60% 차지해도 됨)
- 본문: 제목의 절반 이하 크기 — 확실히 작게
- 강조어: 볼드 or 포인트 컬러로만
- headline font-size: 56~80px (텍스트 길면 44px까지 축소)
- subtext font-size: 26~34px (헤드라인의 절반 이하)
- 리스트 항목 font-size: 28~36px (너무 작으면 가독성 떨어짐)
- ★ 박스/카드/섹션 안의 텍스트도 최소 26px 이상 — 박스 안이라고 폰트를 줄이지 마세요
- ★ 박스 제목(box-title): 최소 32px 이상 — 제목이 작으면 임팩트가 없음
- ★ 번호 리스트(01., 02. 등): 번호 font-size 36~48px(var(--color-primary)), 텍스트 28~36px, 수평 정렬 맞출 것
- ★ 리스트 항목 본문 텍스트: 최소 32px 이상 — 너무 작으면 가독성 떨어짐
- 헤드라인 한 줄 최대 16자. 초과 시 자동 줄바꿈 되도록 max-width 설정
- word-break: keep-all 반드시 적용 (한국어 단어 중간에서 잘리면 안 됨)
- line-height: 1.25~1.35 (1.4 이상은 너무 헐거움, 콤팩트하게)
- 리스트 항목 간격: margin-bottom 16~24px (항목이 고르게 분포되도록)

## 여백 (최소한으로! 콘텐츠가 화면을 채워야 함)
- ★★★ 여백이 너무 많으면 실패 — 콘텐츠가 화면 전체에 고르게 분포해야 함 ★★★
- ❌ 텍스트가 한쪽(상단/중앙)에만 몰리고 나머지가 텅 비는 레이아웃 금지
- ❌ 카드 하단 50% 이상이 빈 공간이면 실패
- 사방 여백: 30~40px (safe-area: 40px 권장)
- ★★★ 왼쪽 여백 최소 60px 이상 — 왼쪽이 답답하면 실패 ★★★
- 텍스트 컨테이너 padding: 40~56px (좌우), 상하는 콘텐츠에 맞게
- 여백보다 콘텐츠 가독성이 우선 — 폰트가 작아지느니 여백을 줄여라
- 리스트 항목 사이 간격: 20~28px (항목들이 화면에 고르게 퍼지도록)
- 카드 내 섹션 간 간격: 20~30px
- ★ 콘텐츠가 적을 때: 폰트를 더 키우거나 간격을 넓혀서 화면을 채워라
- ★ 콘텐츠가 많을 때: 여백을 줄이되 폰트 크기는 유지하라

## ★★★ 콘텐츠 수직 분배 — 상단 몰림 금지 ★★★
- 콘텐츠가 카드 상단에만 몰리고 하단이 비면 실패
- 상단/중단/하단에 콘텐츠를 고르게 분배하세요
- flex 컨테이너에 justify-content: space-between 또는 space-evenly 사용
- 리스트 항목이 있으면 flex: 1 + justify-content: space-evenly로 수직 분포
- 콘텐츠가 적을 때: justify-content: center로 중앙 배치

## ★★★ 아이콘-텍스트 간격 — 맥락에 맞게 조절 ★★★
- 📦 박스/카드 안 (아이콘 위 + 텍스트 아래 세로 배치): margin-bottom 12~16px — 너무 떨어지면 관계가 끊겨 보임
- 📋 인라인 리스트 (아이콘 옆 + 텍스트 가로 배치): margin-right 20~24px — 적당한 간격
- ❌ 박스 안에서 아이콘과 텍스트가 28px 이상 떨어지면 실패 (관계 단절)
- ❌ 인라인에서 아이콘과 텍스트가 12px 이하면 실패 (답답함)
- ★ 아이콘과 텍스트의 수직 정렬 반드시 맞추기: display:flex; align-items:center;

## ★★★ 성과/통계 카드 — 강조가 핵심 ★★★

### 테크닉 A: 장식 아이콘 (Lucide 배경 장식)
- 주제와 관련된 Lucide 아이콘 1~2개를 120~200px 크기로 배치
- opacity: 0.08~0.15 로 배경 장식처럼 깔기 (텍스트를 방해하지 않도록)

### 테크닉 B: CSS 기하학 도형

## <em> 태그 강조 스타일 — ★★★ 강조는 확실하게 ★★★
- <em> 태그가 있으면 반드시 시각적으로 눈에 확 띄어야 합니다
- 강조가 안 보이면 실패 — 주변 텍스트와 확실한 차이를 주세요
- emphasis_style이 highlight → em { background: linear-gradient(to top, var(--color-highlight) 35%, transparent 35%); font-style: normal; font-weight: 900; padding: 2px 6px; margin: 0; display: inline; box-decoration-break: clone; -webkit-box-decoration-break: clone; }
  ⚠️ 절대 규칙: background는 반드시 linear-gradient 방식으로! 글자 하단 35%를 칠하는 형광펜 마커 스타일. 단색 background 사용 금지!
  ⚠️ 하이라이트 색이 배경과 비슷하면 안 보임 → 배경 대비 충분히 다른 색 사용
- emphasis_style이 color → em { color: var(--color-primary); font-weight: 900; font-style: normal; font-size: 110%; }
  ⚠️ 크기도 약간 키워서 강조 효과를 확실히
- emphasis_style이 underline → em { border-bottom: 6px solid var(--color-accent); font-style: normal; font-weight: 900; padding-bottom: 2px; }

## ★★★ 배경-텍스트 대비 적응 규칙 (반드시 지켜야 함) ★★★
배경색에 따라 텍스트·하이라이트·아이콘 색을 반드시 바꿔야 합니다.
같은 CSS를 다크/라이트에 공유하면 가독성이 0이 됩니다.

### 다크 배경일 때 (var(--color-primary), var(--color-text), 짙은 단색)
- 일반 텍스트: #FFFFFF (흰색)
- ★★★ em highlight: em { background: linear-gradient(to top, var(--color-highlight) 35%, transparent 35%); color: #FFFFFF; font-weight: 900; padding: 2px 8px; }
  → 다크 배경에서 var(--color-highlight)가 충분히 밝으면 그대로 사용 (노란색 등)
  → 만약 안 보이면 var(--color-accent) 또는 더 밝은 색으로 교체
  → ★ 핵심: 강조 부분이 주변 텍스트와 확실히 구분되어야 함. 구분 안 되면 실패!
- em color: em { color: var(--color-secondary); font-weight: 900; font-size: 110%; } 또는 em { color: var(--color-accent); }
  → var(--color-primary)는 배경과 같은 색이므로 절대 사용 금지
- 아이콘 색: #FFFFFF 또는 var(--color-accent)
- 서브텍스트: rgba(255,255,255,0.8) (살짝 투명한 흰색)

### 라이트 배경일 때 (var(--color-background), #FFFFFF, 밝은 단색)
- 일반 텍스트: var(--color-text) 또는 #111111
- em highlight: 기본 스타일 유지 (var(--color-highlight) 30% 마커)
- em color: var(--color-primary) — 기본 스타일 유지
- 아이콘 색: var(--color-primary)
- 서브텍스트: var(--color-text) 또는 rgba(0,0,0,0.6)

### 컬러 블록(영역 분할)일 때
- 다크 영역 안의 텍스트 → 다크 규칙 적용
- 라이트 영역 안의 텍스트 → 라이트 규칙 적용
- 한 카드 안에서 영역별로 색을 다르게 적용해야 함

### 자주 발생하는 실수 (절대 금지)
- ❌ 다크 배경 + var(--color-primary) 텍스트 → 배경과 같은 색, 안 보임
- ❌ 다크 배경 + var(--color-highlight) 하이라이트 → 연한 노란색이 짙은 배경에서 안 보임
- ❌ 라이트 배경 + #FFFFFF 텍스트 → 흰 글씨가 흰 배경에서 안 보임
- ❌ 모든 카드에 동일한 em 스타일 → 다크/라이트 카드별로 em 스타일을 분기해야 함

- 배경색: var(--color-background) 또는 #ffffff(흰색)만

- 카드마다 컬러 테마가 달라지면 안 됨 — 10장이 하나의 시리즈

## 카드 타입별 레이아웃 (강제 지정)
- hook → 단색 또는 그라디언트 배경 + 중앙에 초대형 헤드라인. 이미지 사용 시 상단/하단 분리
- problem/empathy → 좌측에 var(--color-primary) 세로 사이드바(8px) + 우측에 텍스트 본문
- data → ★ 구체적 숫자가 반드시 크게 보여야 함 ★
  - stat 숫자를 초대형(72~120px, var(--color-accent))으로 중앙 배치
  - 숫자 없이 라벨(텍스트)만 나열하면 실패 — 반드시 구체적 수치(%,명,점,배 등)를 크게 표시
  - 여러 지표를 보여줄 때: 2x2 그리드로 배치하되, 각 셀 안에 큰 숫자 + 작은 라벨 구조
  - ❌ 카드 박스 안에 라벨만 있고 숫자가 없는 디자인은 실패
  - 예: "의대 합격" → "매년 의대 합격생 배출" + 큰 숫자, "1등급 비율" → "42%" + 설명
- insight → 상단 아이콘 + 중앙 텍스트, 깨끗한 카드형 레이아웃
- solution → ★ 반드시 번호 매긴 스텝형 레이아웃 ★
  - "N단계" 제목이 있으면 각 단계를 CSS 원형 배지 숫자(1, 2, 3...)로 시각 구분
  - 각 단계 사이에 세로 연결선(border-left 또는 ::before) 또는 화살표로 흐름 표현
  - 단계별로 시각적 크기/색상 차이를 줘서 진행감 표현
  - ❌ 모든 단계가 동일한 스타일(같은 크기 화살표 + 텍스트)이면 실패
  - ✅ 각 단계: 좌측 원형 숫자 배지(48px, var(--color-primary)) + 우측 텍스트 + 연결선
- example → 비교 카드(before/after) 또는 인용문 스타일
  - vs 비교 카드일 때: ★ 우리 학원 쪽을 반드시 더 크게/밝게/강조해서 시각적 우위를 보여야 함
  - 우리 학원 컬럼: 배경 var(--color-primary) + 흰색 텍스트 + 체크 아이콘, 폰트 더 크게
  - 일반 학원 컬럼: 회색 배경 + 흐린 텍스트 + 작은 폰트 — 확실히 약하게 표현
  - 문제점 vs 솔루션 비교 카드일 때: ★ 솔루션(우리 학원) 쪽을 더 강조
  - 솔루션 컬럼: 배경 var(--color-primary) 또는 border 강조 + 체크 아이콘 + 폰트 크게

## 배경 이미지가 있을 때 (bg_image_url 존재)
- ★★★ 배경 이미지가 제공되면 반드시 눈에 띄게 크게 사용하세요 ★★★
- 이미지를 작게 축소하거나 안 보이게 처리하면 실패
- ★ 이미지 위에 텍스트 절대 금지. 반투명 오버레이도 금지. rgba 오버레이도 금지 ★
- 이미지 영역과 텍스트 영역을 물리적으로 나누세요
- 이미지는 카드 면적의 40~60%를 차지해야 함

## 배경 이미지가 없을 때 (대부분의 카드)
- ★★★ 배경 이미지가 없으면 단색 배경만 사용하세요 ★★★
- ❌ 절대 금지: 그라데이션 배경 (linear-gradient, radial-gradient 등)
  - 보라색/파란색 그라데이션, 흐린 그라데이션 등 모두 금지
  - 그라데이션은 콘텐츠와 무관한 분위기를 만들고 가독성을 해침
- ✅ 허용: 단색 var(--color-primary), 단색 var(--color-background), 단색 #FFFFFF, 단색 var(--color-text)
카드 타입에 따라 아래 배경 전략을 선택하세요:

### 다크 배경 전략 (임팩트형)
- body 배경: var(--color-primary) 또는 var(--color-text) — 단색만!
- 텍스트: #FFFFFF
- 강조: var(--color-accent) 또는 var(--color-highlight)
- 적합: hook, data, cta 카드

### 라이트 배경 전략 (깔끔형)
- body 배경: var(--color-background) 또는 #FFFFFF
- 텍스트: var(--color-text)
- 포인트: var(--color-primary) (텍스트 강조, 아이콘, 사이드바)
- 적합: insight, solution, summary 카드

### 컬러 블록 전략 (영역 분할형)
- 상단: var(--color-primary) 풀배경 + 흰색 텍스트
- 하단: #FFFFFF + 다크 텍스트
- 또는 좌측 세로 사이드바(var(--color-primary)) + 우측 본문
- 적합: problem, empathy, example 카드

## 10장 시리즈 배경 일관성 규칙
- 다크/라이트 전략을 시리즈 내에서 섞되, 기본 톤은 통일
- 예: 기본 라이트 배경 + hook/cta만 다크 = OK
- 예: 카드마다 다크↔라이트 번갈아가며 = NG (산만)
- 그라데이션은 최소한으로 (linear-gradient 하나, 2색 이내)
- ❌ 금지: 여러 색 그라데이션, 무지개, 매 카드 다른 배경색

## 컬러 사용 요약
| 용도 | 변수 | 비고 |
|------|------|------|
| 배경(다크) | var(--color-primary) | 풀배경 용도 |

HTML을 반환하기 전에 아래 10가지를 스스로 검증하세요:
□ 각 카드 텍스트 3줄 이내인가?
□ 이미지와 텍스트 영역이 분리됐는가? (오버레이 사용 안 했는가?)
□ 컬러가 2가지 이내인가? (primary + background만)
□ 비주얼 포인트가 1개인가?
□ 사방 여백 30~40px인가? (60px 이상은 너무 넓음)
□ 브랜드 바(로고)가 일관되게 들어갔는가?
□ 킬러 카드에 임팩트가 있는가?
□ ★ 모든 콘텐츠가 1080x1350px 안에 들어가는가? (넘치는 텍스트 없는가?)
□ ★ line-height가 1.35 이하인가? 폰트가 충분히 큰가? (headline 56px+, subtext 26px+)
□ ★ 이모지를 사용하지 않았는가? (Lucide 아이콘만 사용했는가? lucide CDN + createIcons() 포함했는가?)
□ ★ 대비 규칙: 헤드라인과 서브텍스트 크기 차이가 2배 이상인가? "중간 크기" 요소가 없는가?
□ ★ 인용문/박스가 있다면: 확실한 스타일인가? (큰따옴표 장식 or 꼬리 달린 말풍선) 애매한 회색 네모가 아닌가?
□ ★ 장식용 아이콘이 있다면: 충분히 큰가? (배경 배지 56px+) "있으나 마나"한 24px 아이콘이 아닌가?
□ ★★ 배경-텍스트 대비: 다크 배경 카드에서 텍스트가 흰색인가? em 하이라이트가 배경과 구분되는 밝은 색인가? 다크 배경에 var(--color-primary) 텍스트를 쓰지 않았는가?
□ ★★ <em> 강조가 확실히 눈에 띄는가? 강조 부분이 주변 텍스트와 시각적으로 확연히 구분되는가? 강조가 안 보이면 실패!
□ ★★ 배경 이미지(bg_image_url)가 제공됐다면 카드 면적의 40~60%를 차지하는가? 이미지가 안 보이거나 너무 작으면 실패!
□ ★★ "N단계" 콘텐츠라면 각 단계가 번호 배지로 시각 구분되는가? 모든 항목이 동일한 스타일이면 실패!
□ ★★ data 카드에 구체적 숫자(%, 명, 점 등)가 크게 표시되는가? 라벨만 있고 숫자가 없으면 실패!
□ ★★ 영어 장식 텍스트가 있는가? "NEWS", "Reality is harsher" 등 불필요한 영어가 하나라도 있으면 실패!
□ ★★ 그라데이션 배경을 사용했는가? 배경 이미지 없는 카드에 그라데이션 사용하면 실패! 단색만 허용!
