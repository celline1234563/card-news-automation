## 카드 10장 구조 규칙

| 카드 | type | 역할 | 사용 템플릿 |
|------|------|------|-----------|
| 1번 | hook | 후킹 — 스크롤 멈추게 | cover-bold |

| 10번 | cta | CTA — 상담 유도 | cta (버튼형) |

## content_brief 작성법 ★★★

content_brief는 이 기획의 핵심입니다. 200~300자로 다음 5가지를 반드시 포함:

1. **핵심 메시지**: 독자에게 전달할 단 한 가지 (한 문장)
2. **근거/데이터**: 리서치에서 찾은 구체적 수치·출처·사례 — 리서치 결과에서 직접 인용
3. **서술 방향**: 톤과 관점 (예: "학부모 입장에서 불안감 자극")
4. **금지 사항**: 이 카드에서 하면 안 되는 것 (예: "아직 해결책 단계 아니므로 학원 언급 금지")
5. **앞뒤 카드 연결**: 이전 카드 감정 → 이 카드 → 다음 카드 전환

## 카드 타입별 추가 필수 필드

### 카드 1번 (hook)
```
"deco_icons": ["Lucide 아이콘1", "아이콘2"]
```
- 리서치의 hook_candidates 중 가장 강력한 것을 선택하세요

### 카드 2번 (problem → speech)
```
"quote_main": "학부모가 실제로 할 법한 말 (30~60자)",
"quote_sub": "짧은 답변/반응 (20~40자)",
"sender": "학부모"
```

### 카드 3번 (problem → info 2×2)
```
"items": [
  { "icon": "이모지", "title": "8자 이내", "desc": "구체적 문제점 20자 이내" },
  ...4개
]
```

### 카드 4번 (data → stat)
```
"stat": "73%",
"stat_label": "20자 이내 설명"
```
- ★ 리서치의 statistics에서 가장 임팩트 있는 수치를 선택

### 카드 5번 (insight → step)
```
"steps": [
  { "title": "10자 이내", "desc": "25자 이내" },
  ...3~4개
]
```

### 카드 6번 (solution → list)
```
"items": ["실행 가능한 구체적 행동 20자 이내", ...5개]
```

### 카드 7번 (solution → compare)
```
"before_title": "일반 학원",
"before_items": "항목1<br>항목2<br>항목3",
"after_title": "{{ACADEMY_NAME}}",
"after_items": "항목1<br>항목2<br>항목3"
```

### 카드 8번 (example → info 2×2)
```
"items": [
  { "icon": "이모지", "title": "8자 이내", "desc": "20자 이내" },
  ...3~4개
]
```
- ★ 리서치의 student_stories나 usable_data_points를 활용

### 카드 9번 (summary → list)
- 카드 6과 중복 금지. 카드 6=학원이 하는 것, 카드 9=학생에게 생기는 변화

### 카드 10번 (cta)
```
"cta_text": "무료 상담 신청하기",
"cta_sub": "카카오 오픈채팅 / 전화 상담 가능"
```

## 공통 필드 (모든 카드)

- **headline**: 20자 이내. `<em>` 태그로 강조. 콜론(:) 금지.
- **subtext**: 40자 이내. 1~2줄.
- **content_brief**: ★★★ 200~300자. 위 5가지 항목 필수.
- **emphasis_style**: "highlight" / "color" / "underline"
- **layout_hint**: 레이아웃 힌트
- **icon**: Lucide 아이콘명
- **image_category**: 실사진 카테고리 또는 null
- **visual_hint**: "graph" / "table" / "photo" / "icon-set" / "before-after" 또는 null
- **focal_point**: 시선이 먼저 가야 할 요소 + 이유
- **emotion**: "위기감" / "공감" / "호기심" / "신뢰" / "안도" / "결심" / "기대"
- **design_brief**: Gemini용 2~3줄 구성 방향

```json
  "research_summary": "리서치 요약 (리서치 결과에서 가져오기)",

      "headline": "...",
      "subtext": "...",
      "content_brief": "200~300자 상세 기술...",

      "layout_hint": "...",
      "icon": "...",
      "image_category": null,
      "visual_hint": null,
      "focal_point": "...",
      "emotion": "...",
      "design_brief": "...",
      "deco_icons": [...]
    },
    ... (정확히 10개)
  ]
}
```
