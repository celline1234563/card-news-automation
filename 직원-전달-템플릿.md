# 직원에게 회의록 전달용 템플릿

> 이 파일은 사용자(원장님 측)가 직원에게 회의록을 보낼 때 그대로 복사해서 쓰는 템플릿입니다.

---

## 1️⃣ 처음 한 번만 (이 가이드가 새로 추가됐을 때)

직원에게 카톡/메일로 한 번 보내세요:

```
VS Code 열고 Claude Code에 아래 한 줄 입력해줘:

업데이트 받아줘
```

→ 직원 컴퓨터에 새 가이드(`EMPLOYEE-GUIDE.md`)가 들어가고, 이후로 회의록 등록 워크플로우가 작동합니다.

---

## 2️⃣ 매번 회의록 보낼 때

아래 블록을 그대로 복사해서 직원에게 보내세요. **`학원:` 줄과 회의록 본문만 채우면 끝.**

```
VS Code 열고 Claude Code에 아래 그대로 붙여넣어:

─────────────
회의록 등록해줘.
학원: 진학

[여기에 회의록 본문을 받은 그대로 붙여넣기]
─────────────
```

### 학원 적는 방법

| 학원 | 적을 단어 |
|------|-----------|
| 진학학원 | `진학` |
| 올인원 수학학원 | `올인원` |
| 톡톡 잉글리쉬 | `톡톡` |

---

## 3️⃣ 직원이 받은 후 자동으로 일어나는 일

1. Claude Code가 회의록 보고 핵심 주제를 한 줄 뽑아 직원에게 확인받음
2. 직원이 OK → 노션에 새 카드 생성 (제목 자동 채번 `[학원 월-번호]`, 상태=`기획 착수`, 제작 포맷=`카드뉴스`)
3. 30초 안에 Railway poller가 잡아서 작은기획(카드 10장) 자동 생성
4. 약 10분 후 노션 카드에 PNG 10장 첨부 완료
5. 직원이 노션에서 PNG 다운로드해서 사용자분께 전달

---

## 4️⃣ 여러 학원이 섞인 회의록일 때

회의록 본문에 학원 두 개 이상이 섞여 있으면, 직원에게 보낼 때 **각 학원별로 나눠서 두세 번 보내**시면 됩니다. 또는 그냥 한 번에 보내도 Claude Code가 학원별로 분할해서 등록합니다(`EMPLOYEE-GUIDE.md`에 그렇게 적혀있음).

---

## 5️⃣ 자주 생기는 문제

| 증상 | 원인 / 대처 |
|------|-------------|
| 직원이 "최신 버전 받아줘" 했는데 에러 | 직원 컴 git 인증 풀렸을 가능성. 사용자분이 한 번 가서 봐주셔야 함 |
| "NOTION_API_KEY 없음" 에러 | 직원 `.env`가 비었거나 손상. 키 다시 전달 필요 |
| 노션엔 카드 생긴 것 같은데 PNG가 안 옴 | Railway poller 멈췄을 수 있음. Railway 대시보드 확인 |
| 같은 회의록을 두 번 등록 | 노션에서 중복 카드 하나 삭제하면 됨 (poller가 두 번 처리해도 결과는 멱등) |
